# 🍽️ 6-can-do-eat (6시 내고향)

> **"우리는 서비스의 안정성과 확장성을 고민하는 백엔드 개발 팀, 6-can-do-eat입니다."**

배달 서비스의 핵심 비즈니스 로직을 설계하며, 단순한 기능 구현을 넘어 탄탄한 인프라와 데이터 정교함을 추구합니다.

### 🚀 What We Do
우리는 **6CanDOEat** 프로젝트를 통해 다음과 같은 기술적 도전들을 해결하고 있습니다:

* **Robust Backend**: Spring Boot, JPA, Spring Security, **Redis**를 활용한 안정적인 배달 도메인 설계
* **Scalable Infrastructure**: GCP(Cloud Run, Cloud SQL)와 Docker를 활용한 클라우드 네이티브 환경 구축
* **Automation**: GitHub Actions를 통한 CI/CD 파이프라인 자동화로 개발 효율성 극대화
* **Advanced Optimization**: 데이터 동시성 제어(**비관적 락**), DB 부하 분산(**Soft Delete 최적화**) 등 성능 고도화

### 👥 Our Members
함께 성장하며 기술적 한계를 극복해 나가는 **Team 6** 구성원입니다:
* **Leader**: 박성우 (주문/결제/AI)
* **Members**:
    * **서동원**: 인증/인가, 운영지역
    * **박준식**: 가게/메뉴, 리뷰
    * **김지민**: 사용자/배송지, 인프라

### 🛠️ Tech Stack
* **Language & Framework**: Java 17, Spring Boot, Spring Security, JPA
* **Database**: PostgreSQL, Redis, Cloud SQL, Memorystore
* **DevOps**: GCP, Docker, GitHub Actions, Terraform
* **Tools**: IntelliJ IDEA, Slack, Notion

---

### 💡 Facts
* 사용자(User)와 주소(Address) 도메인 설계 시 **Soft Delete**와 **Soft Coding** 전략을 깊이 있게 적용하여 유지보수성을 높였습니다.
* 팀명 **6시 내고향**처럼 끈끈한 결속력을 바탕으로, 튜터님 피드백을 통해 인프라 리소스 최적화 및 DB 분할 고도화를 지속적으로 진행 중입니다.
